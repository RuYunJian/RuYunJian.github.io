# iLoboke足球机器人核心任务数学模型

检索日期：2026-05-18
说明：本文只基于网上公开资料和机器人足球常用建模方法整理。iLoboke公开规则明确了场地、SOM平台、定位球/3V3比赛任务和违规约束；部分具体函数名如 getballopp 未找到官方公开数学定义，因此本文给出的是可落地的工程数学模型，而不是官方源码说明。

# 1. 建模基础

# 4. getball 抓球模型

# 4.1 抓球前置点

getball 的目标不是简单冲向球，而是从正确方向接近球，使机器人拿球后可以直接传球或射门。设下一目标点为 ptp_tpt​，例如接球队友或球门空档：

et=pt−pb∥pt−pb∥e_t=\frac{p_t-p_b}{\|p_t-p_b\|} et​=∥pt​−pb​∥pt​−pb​​

抓球前置点：

ppre=pb−ρetp_{\text{pre}}=p_b-\rho e_t ppre​=pb​−ρet​

其中 ρ\rhoρ 是机器人吸球口到球心的安全距离，通常略大于机器人前端半径与球半径之和。

到达时理想朝向：

θpre=atan2⁡(et,y,et,x)\theta_{\text{pre}}=\operatorname{atan2}(e_{t,y},e_{t,x}) θpre​=atan2(et,y​,et,x​)

# 4.2 选择执行抓球机器人的代价

Ropp(i)=max⁡(0,Ti(ppre,θpre)−Topp,min(pb)+m)R_{\text{opp}}(i)= \max(0,T_i(p_{\text{pre}},\theta_{\text{pre}})-T_{\text{opp,min}}(p_b)+m) Ropp​(i)=max(0,Ti​(ppre​,θpre​)−Topp,min​(pb​)+m)

mmm 是安全时间裕度。若己方机器人比对手慢太多，getball 应切换为 getballopp 或防守。

# 5. passball 传球模型

iLoboke省赛要求除点球外，射门前至少完成一次传接配合，并且整个过程要求在15秒内完成4。因此 passball 应建模为“传球点搜索 + 传球力度 + 接球同步 + 拦截风险评估”。

# 5.1 候选传球点

设候选传球点集合为：

P={pk∈Ω∣pk∉Ωforbid, d(pk,pobs)>rsafe}\mathcal{P}=\{p_k\in\Omega \mid p_k\notin\Omega_{\text{forbid}},\ d(p_k,p_{\text{obs}})>r_{\text{safe}}\} P={pk​∈Ω∣pk​∈/Ωforbid​, d(pk​,pobs​)>rsafe​}

可用网格采样、角度采样或预设战术点生成。

# 5.2 接球者到达时间与传球时间

Tr(pk)=Tmove(∥pk−pr∥)+λθ∣wrap⁡(θr−θreceive)∣ωmax⁡T_r(p_k)=T_{\text{move}}(\|p_k-p_r\|) +\lambda_\theta \frac{|\operatorname{wrap}(\theta_r-\theta_{\text{receive}})|}{\omega_{\max}} Tr​(pk​)=Tmove​(∥pk​−pr​∥)+λθ​ωmax​∣wrap(θr​−θreceive​)∣​

ZJUNlict论文中也把传球时间拆成接球机器人到达时间和接球后的缓冲调整时间，并给出类似“缓冲时间可取常数”的做法3。

# 7. getballopp 对手控球/抢截模型

# 7.1 资料边界

公开资料未找到 getballopp 的官方定义。本文将其建模为：当对手更可能先触球或已经控球时，己方机器人执行抢截球、封堵传球线、压迫持球人或退防的任务。

# 14. 可直接实现的伪代码

输入：视觉状态 ball, robots_our, robots_opp, referee_state, time_left

1. 更新球和机器人的滤波状态
2. 计算 T_our, T_opp, P_our, P_opp
3. 若己方已控球：
      计算所有 pass 点的 S_pass
      计算当前 shoot 的 S_shoot 和 canShoot
      若 canShoot 且 S_shoot > S_pass:
          执行 shoot
      否则:
          执行 passball + receiveball
4. 若己方未控球且 P_our 高：
      执行 getball
5. 若对方更可能控球：
      计算 intercept / block / press 三类得分
      选择得分最高且不犯规的 getballopp 子策略
6. 其余机器人执行 defend 或 off-ball running
7. 用角色分配模型重新匹配机器人和任务
8. 将任务目标交给技能层和路径规划层执行

# 16. 结论

这些任务都有可以落地的数学模型。getball 主要是前置点和到达时间最小化；passball 主要是球速反推、传球点搜索和拦截时间约束；receiveball 是球-机器人到达时间匹配；getballopp 可以建模为对手控球概率、截球点、封堵线和压迫点选择；shoot 是有效射门角与拦截预测的融合；defend 和 goalie 是威胁评分与射门线封堵；无球跑位是区域价值函数最大化；角色分配可用匈牙利算法求总代价最小。

比赛中真正有效的做法，是把这些模型拆成若干可调权重的评分函数，然后用比赛日志不断拟合参数。这样既能满足省赛定位球“15秒内传接后射门”的稳定性要求，也能扩展到国赛3V3中的抢球、防守和攻防转换。
