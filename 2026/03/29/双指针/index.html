# 双指针与滑动窗口

# 题目描述

给定一个整数数组 nums 和一个整数 k，请你返回数组中不同的 k-diff 数对的数目。

k-diff 数对定义为一个整数对 (nums[i], nums[j])，需满足以下全部条件：

  1. 0 <= i, j < nums.length

  2. i != j

  3. |nums[i] - nums[j]| == k

双指针是一个解决暴力枚举的技巧，实则应该连算法都算不上，双指针的一般使用前提是数组内部自然有序，也可以后期自主排列

那么就本题而言，假设存在一个数组

int arr={1,2,3,4,5}
//假设我要使得两个数字的相减的差为3
//第一个指针落在1上，第二个落在2上，因为2-1=1，也就是说差值1<3,所以需要2更大，那么右边的指针向右滑动
//同理，如果相减出来的值比如4>3,则说明减的数字太大，那么左边的指针向左滑动即可

# 洛谷 P1638 逛画展（完整题目）

# 题目描述

博览馆正在展出由世上最佳的 m 位画家所画的图画。

游客在购买门票时必须说明两个数字 x 和 y，代表他要看展览中的第 x 幅至第 y 幅画（包含 x, y）之间的所有图画，而门票的价钱就是一张图画一元。

Sept 希望入场后可以看到所有名师的图画（每位画家至少一幅）。当然，他想最小化购买门票的价格。

请求出他购买门票时应选择的 x, y，数据保证一定有解。

若存在多组解，输出 x 最小的那组。

# 输入格式

第一行两个整数 n, m，分别表示博览馆内的图画总数及画家的数量。

第二行包含 n 个整数 a₁, a₂, ..., aₙ，代表第 i 幅画的名师编号（编号介于 1~m 之间）。

# 解答

这是典型的 滑动窗口（双指针） 题目：

  1. 用 l, r 维护窗口 [l, r]，保证窗口内包含所有 m 种画家
  2. r 右移扩大窗口，直到满足条件
  3. l 尽可能右移缩小窗口，同时保持条件满足，更新最优解
  4. 统计画家出现次数，用计数器判断是否集齐所有 m 种
