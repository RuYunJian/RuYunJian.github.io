# AI 专业名词解释

这份文档用于快速理解 AI、LLM、API、Agent、MCP、Skill 等相关概念。术语按主题分组，适合初学者、产品经理、开发者和需要与 AI 系统协作的人阅读。

# 1. 基础概念

# AI

AI 是 Artificial Intelligence 的缩写，中文通常称为“人工智能”。它泛指让机器具备感知、理解、推理、生成、决策或执行任务能力的技术体系。

常见应用包括：

  • 文本生成
  • 图像生成
  • 语音识别
  • 智能客服
  • 推荐系统
  • 自动驾驶
  • 数据分析
  • 代码生成

# Machine Learning

Machine Learning，简称 ML，中文是“机器学习”。它是 AI 的一个重要分支，核心思想是让模型从数据中学习规律，而不是完全依赖人工编写规则。

例如：给模型大量图片和标签，模型可以学习识别猫、狗、汽车等对象。

# Deep Learning

Deep Learning，中文是“深度学习”。它是机器学习的一类方法，通常使用多层神经网络来处理复杂数据。

深度学习常用于：

  • 图像识别
  • 语音识别
  • 自然语言处理
  • 大模型训练
  • 自动驾驶感知

# Model

Model，中文是“模型”。在 AI 中，模型可以理解为经过训练后具备某种能力的程序。

例如：

  • 文本模型可以回答问题、写文章、写代码
  • 图像模型可以生成图片
  • 语音模型可以识别或合成声音
  • 多模态模型可以同时理解文本、图片、音频、视频

# LLM

LLM 是 Large Language Model 的缩写，中文是“大语言模型”。它是一类能够理解和生成自然语言的 AI 模型。

常见能力包括：

  • 对话
  • 总结
  • 翻译
  • 写作
  • 代码生成
  • 信息抽取
  • 推理
  • 工具调用

# Multimodal Model

Multimodal Model，中文是“多模态模型”。它可以处理多种类型的信息，例如文本、图片、音频、视频、表格等。

例如：用户上传一张截图并提问，模型能够读取图片内容并给出解释。

# Generative AI

Generative AI，中文是“生成式 AI”。它指能够生成新内容的 AI，例如文字、图片、音乐、视频、代码等。

典型例子：

  • 生成文章
  • 生成海报
  • 生成 PPT
  • 生成程序代码
  • 生成短视频脚本

# 2. Token 与上下文

# Token

Token 是 AI 模型处理文本时的基本单位。它不一定等于一个字、一个词或一个字符，而是模型分词后得到的小片段。

例如：

  • 英文中，一个单词可能是一个 token，也可能被拆成多个 token
  • 中文中，一个汉字、词语或标点都可能成为 token
  • 空格、换行、标点也可能被计算为 token

Token 影响：

  • 输入长度
  • 输出长度
  • 费用计算
  • 模型能记住的上下文范围
  • 接口调用速度

# Tokenization

Tokenization，中文可称为“分词”或“标记化”。它是把文本转换为 token 的过程。

例如一句话：

I love AI.

模型不会直接按人类阅读方式理解，而是先转换成一组 token，再进行计算。

# Input Token

Input Token 是用户输入给模型的 token，包括问题、上下文、系统指令、历史对话、工具返回内容等。

输入越长，消耗的 token 越多，成本通常也越高。

# Output Token

Output Token 是模型生成回答时产生的 token。

例如模型写了一篇 1000 字文章，这些生成内容就会被计算为输出 token。

# Context Window

Context Window，中文常称为“上下文窗口”。它表示模型一次调用中最多能处理多少 token。

上下文窗口越大，模型能同时参考的信息越多。

例如：

  • 短上下文模型适合简单问答
  • 长上下文模型适合阅读长文档、分析代码库、处理合同、总结会议记录

# Max Tokens

Max Tokens 通常表示一次调用中允许模型最多生成多少 token。

如果设置过小，模型回答可能被截断；如果设置过大，成本和响应时间可能增加。

# 3. Prompt 与消息角色

# Prompt

Prompt，中文常称为“提示词”或“提示”。它是用户给 AI 的输入指令，用来告诉模型要完成什么任务。

好的 prompt 通常包含：

  • 目标
  • 背景
  • 输入材料
  • 输出格式
  • 约束条件
  • 示例

# System Prompt

System Prompt 是系统级指令，通常用于定义模型的身份、行为边界、回答风格和优先规则。

例如：

你是一名严谨的法律助理，请用简洁、准确的语言回答。

# User Message

User Message 是用户发给模型的内容，也就是用户问题或任务说明。

# Assistant Message

Assistant Message 是模型生成的回复内容。

# Developer Message

Developer Message 是开发者给模型的中高优先级指令，通常用于控制应用中的模型行为，例如输出格式、安全规则、工具使用方式等。

# Prompt Engineering

Prompt Engineering，中文是“提示词工程”。它是设计、优化和测试 prompt 的方法，使模型更稳定、更准确地完成任务。

常见技巧包括：

  • 明确角色
  • 指定输出格式
  • 提供示例
  • 拆分复杂任务
  • 说明边界条件
  • 要求模型先分析再输出

# Few-shot Prompting

Few-shot Prompting 是在 prompt 中提供少量示例，让模型模仿示例完成类似任务。

# Zero-shot Prompting

Zero-shot Prompting 是不给示例，直接让模型完成任务。

# 4. API 与开发接入

# API

API 是 Application Programming Interface 的缩写，中文是“应用程序编程接口”。它允许一个程序调用另一个服务的能力。

在 AI 场景中，API 通常用于：

  • 调用大模型生成文本
  • 调用图像生成模型
  • 调用语音识别服务
  • 上传文件让模型分析
  • 让模型调用工具或函数

# API Key

API Key 是访问 API 服务的密钥。它相当于一把“访问凭证”，用于证明调用者有权限使用某个服务。

注意事项：

  • 不要把 API Key 写进公开代码仓库
  • 不要截图或公开分享 API Key
  • 生产环境应使用环境变量或密钥管理服务保存
  • 如果泄露，应立即删除或轮换

# Key

Key 在不同语境中含义不同。在 AI 开发中，通常指 API Key、访问密钥、加密密钥或配置项中的键名。

常见含义：

  • API Key：访问接口的凭证
  • Secret Key：敏感密钥
  • Public Key：公钥
  • Private Key：私钥
  • Config Key：配置字段名

# Endpoint

Endpoint，中文可称为“接口地址”或“服务端点”。它是 API 请求发送到的 URL。

例如：

https://api.example.com/v1/chat/completions

# Request

Request 是“请求”。当程序调用 AI API 时，会向服务端发送 request，其中通常包含模型名称、用户输入、参数、文件或工具配置。

# Response

Response 是“响应”。它是 API 返回的结果，通常包含模型生成内容、状态信息、错误信息、token 用量等。

# SDK

SDK 是 Software Development Kit 的缩写，中文是“软件开发工具包”。它封装了底层 API 调用，让开发者更方便地使用某个平台。

例如使用 SDK 时，开发者可能只需要写几行代码就能调用模型。

# Rate Limit

Rate Limit 是“速率限制”。它限制一定时间内最多可以调用多少次 API，或者最多可以消耗多少 token。

常见限制包括：

  • 每分钟请求数
  • 每分钟 token 数
  • 每天调用额度
  • 并发请求数量

# Quota

Quota 是“配额”。它表示账户、项目或服务在某个周期内可使用的资源上限。

# Billing

Billing 是“计费”。AI 服务通常按照输入 token、输出 token、图片数量、音频时长、模型类型或工具使用量计费。

# 5. 模型调用参数

# Temperature

Temperature 是控制模型输出随机性的参数。

一般理解：

  • 数值低：回答更稳定、更保守
  • 数值高：回答更多样、更有创造性

常见用法：

  • 事实问答、代码、数据分析：使用较低 temperature
  • 创意写作、头脑风暴：可以使用较高 temperature

# Top P

Top P 是另一种控制输出随机性的参数，也叫 nucleus sampling。

它控制模型在生成下一个 token 时，从概率累计达到某个阈值的候选集合中选择。

# Stop Sequence

Stop Sequence 是“停止序列”。当模型生成内容遇到指定字符串时，会停止继续生成。

# Seed

Seed 是随机种子。某些模型支持设置 seed，使输出更容易复现。

注意：即使设置 seed，模型服务更新或参数变化后，也不一定完全一致。

# Streaming

Streaming 是“流式输出”。模型不是等完整回答生成完才返回，而是一边生成一边返回。

适合：

  • 聊天应用
  • 长文本生成
  • 需要更快首字响应的产品

# 6. Agent、Tool 与函数调用

# Agent

Agent，中文常称为“智能体”。它不仅能回答问题，还能根据目标规划步骤、调用工具、读取文件、执行操作，并根据结果继续调整。

简单理解：

  • 普通聊天模型：主要负责回答
  • Agent：可以围绕目标持续行动

例如一个代码 Agent 可以：

  • 阅读项目文件
  • 修改代码
  • 运行测试
  • 分析错误
  • 继续修复

# Tool

Tool 是“工具”。它是模型可以调用的外部能力。

常见工具包括：

  • 搜索网页
  • 读取文件
  • 写入文件
  • 查询数据库
  • 调用 API
  • 执行代码
  • 生成图片

# Function Calling

Function Calling，中文常称为“函数调用”。它允许模型按照开发者定义的函数结构输出参数，由程序执行对应函数。

例如用户说：

帮我查一下今天上海天气。

模型可以生成类似：

{
  "function": "get_weather",
  "arguments": {
    "city": "Shanghai",
    "date": "today"
  }
}

然后应用程序调用天气接口并把结果返回给模型。

# Tool Calling

Tool Calling 是更宽泛的“工具调用”。它不只包括函数，还可以包括搜索、文件系统、数据库、浏览器、代码执行器等工具。

# Planner

Planner 是“规划器”。在 Agent 系统中，它负责把复杂目标拆解成多个步骤。

例如：

目标：分析竞品网站
步骤：
1. 收集竞品列表
2. 抓取官网信息
3. 比较功能、价格、定位
4. 总结机会点

# Executor

Executor 是“执行器”。它负责执行规划器生成的步骤，例如调用搜索工具、运行代码、写文件等。

# Memory

Memory 是“记忆”。它用于让 AI 系统在多次对话或长期使用中保留信息。

常见类型：

  • 短期记忆：当前对话上下文
  • 长期记忆：用户偏好、项目背景、历史任务
  • 外部记忆：数据库、向量库、文件系统

# 7. Skill 与插件

# Skill

Skill，中文可理解为“技能”。在 AI Agent 系统中，Skill 通常是一组专门的操作说明、流程、脚本或资源，用来让 AI 更擅长完成某类任务。

例如：

  • 数据分析 Skill
  • 写作 Skill
  • 做 PPT Skill
  • 安全审计 Skill
  • Excel 处理 Skill
  • 网页搜索 Skill

一个 Skill 可能包含：

  • 使用规则
  • 工作流程
  • 示例
  • 模板
  • 脚本
  • 参考资料

# Plugin

Plugin，中文是“插件”。插件通常是一组可安装的扩展能力，可以为 AI 应用增加工具、技能、界面或外部服务连接能力。

Skill 更偏“让 AI 学会一种工作方式”，Plugin 更偏“给系统安装一组能力”。

# Workflow

Workflow，中文是“工作流”。它表示完成某个任务的一套固定步骤。

例如“会议纪要工作流”可能包括：

  • 读取会议记录
  • 提取议题
  • 总结决策
  • 提取行动项
  • 标注负责人和截止日期

# Automation

Automation，中文是“自动化”。它指让系统在特定时间、条件或事件触发时自动执行任务。

例如：

  • 每周生成周报
  • 每天早上汇总待办
  • 每小时检查网站状态
  • 定期扫描安全风险

# 8. MCP 相关概念

# MCP

MCP 是 Model Context Protocol 的缩写，中文可称为“模型上下文协议”。它是一种让 AI 应用与外部工具、数据源、服务进行标准化连接的协议。

简单理解：MCP 像是 AI 应用和外部世界之间的统一连接标准。

通过 MCP，模型可以更容易地访问：

  • 文件
  • 数据库
  • 代码仓库
  • 浏览器
  • 企业系统
  • 搜索服务
  • 本地工具

# MCP Server

MCP Server 是提供能力的一端。它把某些工具、资源或数据源暴露给 AI 应用使用。

例如：

  • 文件系统 MCP Server
  • GitHub MCP Server
  • 数据库 MCP Server
  • 浏览器 MCP Server

# MCP Client

MCP Client 是连接 MCP Server 的客户端。通常由 AI 应用、编辑器、Agent 平台或桌面应用承担。

# MCP Host

MCP Host 是运行和管理 MCP Client 的宿主应用。

例如一个 AI 桌面应用可以作为 Host，负责连接多个 MCP Server，并把可用工具提供给模型。

# MCP Resource

MCP Resource 是 MCP Server 提供的资源，例如文件内容、数据库 schema、文档片段、项目配置等。

# MCP Tool

MCP Tool 是 MCP Server 暴露出来的可执行操作，例如查询数据库、读取文件、创建 issue、搜索文档等。

# MCP Prompt

MCP Prompt 是 MCP Server 提供的可复用提示模板，用于帮助模型完成特定任务。

# 9. RAG、Embedding 与向量数据库

# RAG

RAG 是 Retrieval-Augmented Generation 的缩写，中文常称为“检索增强生成”。

它的核心流程是：

  1. 用户提问
  2. 系统从知识库中检索相关资料
  3. 把资料连同问题一起交给模型
  4. 模型基于资料生成回答

RAG 常用于：

  • 企业知识库问答
  • 文档助手
  • 客服机器人
  • 法律、医疗、金融资料查询
  • 私有数据问答

# Retrieval

Retrieval 是“检索”。它指从数据库、搜索引擎、向量库或文件系统中找到与问题相关的信息。

# Embedding

Embedding，中文常称为“嵌入”或“向量表示”。它是把文本、图片、音频等内容转换成一串数字向量。

这些向量可以表示内容之间的语义相似度。

例如：

  • “苹果手机”
  • “iPhone”
  • “Apple mobile phone”

它们文字不同，但语义接近，向量距离也可能较近。

# Vector

Vector 是“向量”。在 AI 中，它通常是一组数字，用来表示文本、图片、音频等内容的特征。

# Vector Database

Vector Database，中文是“向量数据库”。它专门用于存储和检索向量数据。

常见用途：

  • 语义搜索
  • 文档问答
  • 推荐系统
  • 相似图片搜索
  • RAG 知识库

# Similarity Search

Similarity Search 是“相似度搜索”。它根据向量之间的距离或相似度，找出最接近的内容。

# Chunk

Chunk 是“文本块”或“分片”。在 RAG 中，长文档通常会被切分成多个 chunk，再分别进行 embedding 和检索。

切分过大可能导致检索不精准；切分过小可能丢失上下文。

# Reranking

Reranking 是“重排序”。在初步检索出一批候选内容后，系统会再次评估这些内容与问题的相关性，并重新排序。

# 10. 训练、微调与推理

# Training

Training 是“训练”。它指使用大量数据调整模型参数，使模型学习某种能力。

# Pretraining

Pretraining 是“预训练”。大模型通常先在海量通用数据上训练，学习语言、知识和推理基础。

# Fine-tuning

Fine-tuning 是“微调”。它是在已有模型基础上，用特定数据进一步训练，使模型更适合某个场景。

例如：

  • 客服话术
  • 法律文书
  • 医疗问答
  • 企业内部风格
  • 特定代码规范

# Supervised Fine-tuning

Supervised Fine-tuning，简称 SFT，中文是“监督微调”。它使用人工标注的输入输出样本来训练模型。

# RLHF

RLHF 是 Reinforcement Learning from Human Feedback 的缩写，中文常称为“基于人类反馈的强化学习”。

它通过人类偏好数据帮助模型生成更符合人类期望的回答。

# Inference

Inference，中文是“推理”或“推断”。它指模型已经训练好之后，接收输入并生成输出的过程。

用户日常调用 AI 写作、问答、生成代码，大多属于 inference。

# Parameters

Parameters 是“参数”。在模型中，它们是经过训练得到的数值，用来决定模型如何处理输入和生成输出。

模型参数越多，通常表示容量越大，但并不一定代表在所有任务上都更好。

# Weights

Weights 是“权重”。它是模型参数的一种说法，指神经网络中用于计算的数值。

# 11. 评估与质量

# Evaluation

Evaluation，简称 Eval，中文是“评估”。它用于测试模型或 AI 系统在某些任务上的表现。

评估指标可能包括：

  • 准确率
  • 召回率
  • 幻觉率
  • 响应速度
  • 成本
  • 用户满意度
  • 安全性

# Benchmark

Benchmark 是“基准测试”。它是一组标准化任务，用于比较不同模型或系统的能力。

# Hallucination

Hallucination，中文常称为“幻觉”。它指模型生成看似合理但实际错误、无来源或捏造的内容。

减少幻觉的方法：

  • 使用 RAG 提供可靠资料
  • 要求模型引用来源
  • 限制模型只基于给定材料回答
  • 对关键结果做人工审核
  • 使用工具验证事实

# Grounding

Grounding，中文可称为“依据化”或“落地到事实”。它指让模型回答建立在明确数据、文档、工具结果或真实上下文上。

# Guardrail

Guardrail，中文是“护栏”。它是限制模型行为、降低风险的一套规则或系统。

例如：

  • 拒绝危险请求
  • 过滤敏感信息
  • 限制输出格式
  • 防止泄露隐私
  • 对高风险领域增加人工审核

# 12. 安全、隐私与权限

# Secret

Secret 是“机密信息”。它包括 API Key、数据库密码、访问令牌、私钥等。

Secret 不应出现在：

  • 公共代码仓库

# 双指针与滑动窗口

# 题目描述

给定一个整数数组 nums 和一个整数 k，请你返回数组中不同的 k-diff 数对的数目。

k-diff 数对定义为一个整数对 (nums[i], nums[j])，需满足以下全部条件：

  1. 0 <= i, j < nums.length

  2. i != j

  3. |nums[i] - nums[j]| == k

双指针是一个解决暴力枚举的技巧，实则应该连算法都算不上，双指针的一般使用前提是数组内部自然有序，也可以后期自主排列

那么就本题而言，假设存在一个数组

int arr={1,2,3,4,5}
//假设我要使得两个数字的相减的差为3
//第一个指针落在1上，第二个落在2上，因为2-1=1，也就是说差值1<3,所以需要2更大，那么右边的指针向右滑动
//同理，如果相减出来的值比如4>3,则说明减的数字太大，那么左边的指针向左滑动即可

# 洛谷 P1638 逛画展（完整题目）

# 题目描述

博览馆正在展出由世上最佳的 m 位画家所画的图画。

游客在购买门票时必须说明两个数字 x 和 y，代表他要看展览中的第 x 幅至第 y 幅画（包含 x, y）之间的所有图画，而门票的价钱就是一张图画一元。

Sept 希望入场后可以看到所有名师的图画（每位画家至少一幅）。当然，他想最小化购买门票的价格。

请求出他购买门票时应选择的 x, y，数据保证一定有解。

若存在多组解，输出 x 最小的那组。

# 输入格式

第一行两个整数 n, m，分别表示博览馆内的图画总数及画家的数量。

第二行包含 n 个整数 a₁, a₂, ..., aₙ，代表第 i 幅画的名师编号（编号介于 1~m 之间）。

# 解答

这是典型的 滑动窗口（双指针） 题目：

  1. 用 l, r 维护窗口 [l, r]，保证窗口内包含所有 m 种画家
  2. r 右移扩大窗口，直到满足条件
  3. l 尽可能右移缩小窗口，同时保持条件满足，更新最优解
  4. 统计画家出现次数，用计数器判断是否集齐所有 m 种

# set

**集合（set）**是一个内部自动有序且不含重复元素的容器，它可以在需要删除重复元素的情况下大放异彩，节省时间，减少思维量。

set 就是关键字的简单集合，当只是想知道一个值是否存在时，set 是最有用的。set 内部采用的是一种非常高效的平衡检索二叉树 ：红黑树（Red-Black Tree），也称为 RB 树，RB 树的统计性能要好于一般平衡二叉树。在 set 中每个元素的值都唯一，而且系统能根据元素的值自动进行排序，set 中元素的值不能直接被改变。

标准库提供 set 关联容器分为：

按关键字有序保存元素：set（关键字即值，即只保存关键字的容器）、multiset（关键字可重复出现的 set）；
无序集合：unordered_set（用哈希函数组织的 set）、unordered_multiset（用哈希函数组织的 set，关键字可以重复出现）

# set 容器内的元素访问

注意：除了 vector 和 string 以外的 STL 容器都不支持 *(it + i) 的访问方式

  • **find()：**find(value) 返回 set 中 value 所对应的迭代器，即 value 的指针（地址），如果没找到则返回 end()

        #include <iostream>
    #include <set>
    using namespace std;
    int main() 
    {
        set<int> st;
        for (int i = 1; i <= 3; i++)
        {
            st.insert(i);
        }
 
        set<int>::iterator it = st.find(2); // 在set中查找2，返回其迭代器
        cout << *it << endl;
 
        // 以上可以直接写成
        cout << *(st.find(2)) << endl;
        return 0;
    }
 
    // 输出结果
    2
    2

  • erase()：可以删除单个元素或删除一个区间内的所有元素

    1. st.erase(it)，其中 it 为所需要删除元素的迭代器。时间复杂度为 O(1)，可以结合 find() 函数来使用。
    2. st.erase(value)，其中 value 为所需要删除元素的值。时间复杂度为 O(logN)，N 为 set 内的元素个数。
        #include <iostream>
    #include <set>
    using namespace std;
 
    int main()
    {
        set<int> st;
        st.insert(100);
        st.insert(200);
        st.insert(100);
        st.insert(300);
        st.insert(500);
 
        // 删除单个元素
        st.erase(st.find(100)); // 利用find()函数找到100,然后用erase删除它
        st.erase(200);          // 删除值为200的元素
        for (set<int>::iterator it = st.begin(); it != st.end(); it++)
        {
            cout << *it << " ";
        }
        return 0;
    }
 
    // 输出结果
    300 500

  • **equal_range()：**返回一对迭代器，分别表示第一个大于或等于给定关键值的元素和第一个大于给定关键值的元素。这个返回值是一个 pair 类型，第一个是键的 lower_bound，第二个是键的 upper_bound。如果这一对定位器中哪个返回失败，则返回 end()

        #include <iostream>
    #include <set>
    using namespace std;
 
    int main()
    {
        set<int> s;
        set<int>::iterator it;
        for (int i = 1 ; i <= 5; ++i)
        {
            s.insert(i);
        }
        for (it = s.begin() ; it != s.end() ; ++it)
        {
            cout << *it << " ";
        }
        cout << endl;
        pair<set<int>::const_iterator, set<int>::const_iterator> pr;
        pr = s.equal_range(3);
        cout << "第一个大于等于3的数是："<< *pr.first << endl;
        cout << "第一个大于3的数是："<< *pr.second <<endl;
        return 0;
    }
 
    // 输出结果
    1 2 3 4 5
    第一个大于等于3的数是：3
    第一个大于3的数是：4

# Vector

向量（Vector）是一个封装了动态大小数组的顺序容器（Sequence Container）,跟任意其它类型容器一样，它能够存放各种类型的对象。可以简单的认为，向量是一个能够存放任意类型的动态数组

Vector的数据安排以及操作方式，与array(数组)非常相似，两者的唯一差别在于空间的运用的灵活性。

  • Array是静态空间，一旦配置了就不能改变，要换大一点或者小一点的空间，可以，一切琐碎得由自己来，首先配置一块新的空间，然后将旧空间的数据搬往新空间，再释放原来的空间。
  • Vector是动态空间，随着元素的加入，它的内部机制会自动扩充空间以容纳新元素。因此vector的运用对于内存的合理利用与运用的灵活性有很大的帮助，我们再也不必害怕空间不足而一开始就要求一个大块头的array（数组）了
  • Vector的实现技术，关键在于其对大小的控制以及重新配置时的数据移动效率，一旦vector旧空间满了，如果客户每新增一个元素，vector内部只是扩充一个元素的空间，实为不智，因为所谓的扩充空间(不论多大)，一如刚所说，是”配置新空间-数据移动-释放旧空间”的大工程,时间成本很高，应该加入某种未雨绸缪的考虑，稍后我们便可以看到vector的空间配置策略

begin和end

通过begin函数可以得到容器中第一个元素的正向迭代器，通过end函数可以得到容器中最后一个元素的后一个位置的正向迭代器。

#include <iostream>
#include <vector>
using namespace std;
 
int main()
{
	vector<int> v(10, 2);
	//正向迭代器遍历容器
	vector<int>::iterator it = v.begin();
	while (it != v.end())
	{
		cout << *it << " ";
		it++;
	}
	cout << endl;
	return 0;
}

rbegin和rend

通过rbegin函数可以得到容器中最后一个元素的反向迭代器，通过rend函数可以得到容器中第一个元素的前一个位置的反向迭代器。

#include <iostream>
#include <vector>
using namespace std;
 
int main()
{
	vector<int> v(10, 2);
	//反向迭代器遍历容器
	vector<int>::reverse_iterator rit = v.rbegin();
	while (rit != v.rend())
	{
		cout << *rit << " ";
		rit++;
	}
	cout << endl;
	return 0;
}

# map简介

map是STL的一个关联容器，它提供一对一的hash。

第一个可以称为关键字(key)，每个关键字只能在map中出现一次；
第二个可能称为该关键字的值(value)；map以模板(泛型)方式实现，可以存储 任意类型的数据，包括使用者自定义的数据类型。Map主要用于资料一对一映射(one-to-one)的情況，map內部的实现自建一颗红黑树，这颗树具有对数据自动排序的功能。在map内部所有的数据都是有序的，后边我们会见识到有序的好处。比如一个班级中，每个学生的学号跟他的姓名就存在著一对一映射的关系。

以上三种用法，虽然都可以实现数据的插入，但是它们是有区别的，当然了第一种和第二种在效果上是完成一样的，用insert函数插入数据，在数据的 插入上涉及到集合的唯一性这个概念，即当map中有这个关键字时，insert操作是不能在插入数据的，但是用数组方式就不同了，它可以覆盖以前该关键字对 应的值，用程序说明如下：
```cpp
    mapStudent.insert(map<int, string>::value_type (001, "student_one"));
     
    mapStudent.insert(map<int, string>::value_type (001, "student_two"));

上面这两条语句执行后，map中001这个关键字对应的值是“student_one”，第二条语句并没有生效，那么这就涉及到我们怎么知道insert语句是否插入成功的问题了，可以用pair来获得是否插入成功，程序如下
```cpp
    // 构造定义，返回一个pair对象
    pair<iterator,bool> insert (const value_type& val);
     
    pair<map<int, string>::iterator, bool> Insert_Pair;
     
    Insert_Pair = mapStudent.insert(map<int, string>::value_type (001, "student_one"));
     
    if(!Insert_Pair.second)
        cout << ""Error insert new element" << endl;

我们通过pair的第二个变量来知道是否插入成功，它的第一个变量返回的是一个map的迭代器，如果插入成功的话Insert_Pair.second应该是true的，否则为false。

# hash_map原理简介
这是一节让你深入理解hash_map的介绍，如果你只是想囫囵吞枣，不想理解其原理，你倒是可以略过这一节，但我还是建议你看看，多了解一些没有坏处。

hash_map基于hash table（哈希表）。 哈希表最大的优点，就是把数据的存储和查找消耗的时间大大降低，几乎可以看成是常数时间；而代价仅仅是消耗比较多的内存。然而在当前可利用内存越来越多的情况下，用空间换时间的做法是值得的。另外，编码比较容易也是它的特点之一。

其基本原理是：使用一个下标范围比较大的数组来存储元素。可以设计一个函数（哈希函数，也叫做散列函数），使得每个元素的关键字都与一个函数值（即数组下标，hash值）相对应，于是用这个数组单元来存储这个元素；也可以简单的理解为，按照关键字为每一个元素“分类”，然后将这个元素存储在相应“类”所对应的地方，称为桶。

但是，不能够保证每个元素的关键字与函数值是一一对应的，因此极有可能出现对于不同的元素，却计算出了相同的函数值，这样就产生了“冲突”，换句话说，就是把不同的元素分在了相同的“类”之中。 总的来说，“直接定址”与“解决冲突”是哈希表的两大特点。

hash_map，首先分配一大片内存，形成许多桶。是利用hash函数，对key进行映射到不同区域（桶）进行保存。其插入过程是：

得到key
通过hash函数得到hash值
得到桶号(一般都为hash值对桶数求模)
存放key和value在桶内。
其取值过程是:

得到key
通过hash函数得到hash值
得到桶号(一般都为hash值对桶数求模)
比较桶的内部元素是否与key相等，若都不相等，则没有找到。
取出相等的记录的value。
hash_map中直接地址用hash函数生成，解决冲突，用比较函数解决。这里可以看出，如果每个桶内部只有一个元素，那么查找的时候只有一次比较。当许多桶内没有值时，许多查询就会更快了(指查不到的时候).

由此可见，要实现哈希表, 和用户相关的是：**hash函数**和**比较函数**。这两个参数刚好是我们在使用hash_map时需要指定的参数。


例子
```cpp
    #include <hash_map>
    #include <string>
    using namespace std;
    int main(){
            hash_map<int, string> mymap;
            mymap[9527]="唐伯虎点秋香";
            mymap[1000000]="百万富翁的生活";
            mymap[10000]="白领的工资底线";
            ...
            if(mymap.find(10000) != mymap.end()){
                    ...
            }

你没有指定hash函数和比较函数的时候，你会有一个缺省的函数，看看hash_map的声明，你会更加明白。下面是SGI STL的声明：
```cpp
    template <class _Key, class _Tp, class _HashFcn = hash<_Key>,
    class _EqualKey = equal_to<_Key>,
    class _Alloc = __STL_DEFAULT_ALLOCATOR(_Tp) >
    class hash_map
    {
            ...
    }

也就是说，在上例中，有以下等同关系：
```cpp
    hash_map<int, string> mymap;
    //等同于:
    hash_map<int, string, hash<int>, equal_to<int> > mymap;

## hash_map 的hash函数
hash< int>到底是什么样子？看看源码:
```cpp
    struct hash<int> {
            size_t operator()(int __x) const { return __x; }
    };

原来是个函数对象。在SGI STL中，提供了以下hash函数：
```cpp
    struct hash<char*>
    struct hash<const char*>
    struct hash<char> 
    struct hash<unsigned char> 
    struct hash<signed char>
    struct hash<short>
    struct hash<unsigned short> 
    struct hash<int> 
    struct hash<unsigned int>
    struct hash<long> 
    struct hash<unsigned long>

也就是说，如果你的key使用的是以上类型中的一种，你都可以使用缺省的hash函数。当然你自己也可以定义自己的hash函数。对于自定义变量，你只能如此，例如对于string，就必须自定义hash函数。例如：
```cpp
    struct str_hash{
            size_t operator()(const string& str) const
            {
                    unsigned long __h = 0;
                    for (size_t i = 0 ; i < str.size() ; i ++)
                    __h = 5*__h + str[i];
                    return size_t(__h);
            }
    };

    //如果你希望利用系统定义的字符串hash函数，你可以这样写：
    struct str_hash{
            size_t operator()(const string& str) const
            {
                    return __stl_hash_string(str.c_str());
            }
    };

在声明自己的哈希函数时要注意以下几点：

1、使用struct，然后重载operator().
2、返回是size_t
3、参数是你要hash的key的类型。
4、函数是const类型的。

现在可以对开头的string 进行哈希化了 . 直接替换成下面的声明即可：
```cpp
    map<string, string> namemap; 
    //改为：
    hash_map<string, string, str_hash> namemap;

struct binary_function {
        typedef _Arg1 first_argument_type;
        typedef _Arg2 second_argument_type;
        typedef _Result result_type;

};

# 什么时候需要用hash_map，什么时候需要用map

总体来说，hash_map 查找速度会比map快，而且查找速度基本和数据数据量大小，属于常数级别;而map的查找速度是log (n)级别。并不一定常数就比log(n)小，hash还有hash函数的耗时，明白了吧，如果你考虑效率，特别是在元素达到一定数量级时，考虑考虑hash_map。但若你对内存使用特别严格，希望程序尽可能少消耗内存，那么一定要小心，hash_map可能会让你陷入尴尬，特别是当你的hash_map对象特别多时，你就更无法控制了，而且hash_map的构造速度较慢。

#include <vector>
using namespace std;

class Solution {
public:
    int uniquePaths(int m, int n) 
    {
        vector<vector<int>> dp(m, vector<int>(n, 0));
        for (int i = 0; i < m; ++i) 
        {
            dp[i][0] = 1;
        }
        for (int j = 0; j < n; ++j) 
        {
            dp[0][j] = 1;
        }
        for (int i = 1; i < m; ++i) 
        {
            for (int j = 1; j < n; ++j) 
            {
                dp[i][j] = dp[i-1][j] + dp[i][j-1];
            }
        }
        return dp[m-1][n-1];
    }
};

# 一.string类型

1.求长度：arr.length()或者是arr.size()

2.字符串的连接可以直接使用+号，形如c=a+b（eg：a=hello,b=world,c=a+b=hello world）

# 二.for循环与auto迭代器

1.auto可以转化为任意数据类型(eg:string a="abc";auto b=a;)

2.定义一个数组arr[]，有如下语法：

int arr[5]={1,2,3,4,5}
for(auto x:arr)
{
    cout<<x<<endl
}

可以依次打印数组中的元素，输出结果就是12345，代替了三分号的步骤

# 三,引用

引用就是取别名 ，具体如下

int a=10;
int &p=a;
P += 5;
cout<<a<<endl;

此时的输出结果为a=15,也就是说P在发生改变时a也会同时发生改变，就是相当于a的别名为p

# 四.常用库函数

I.reverse翻转

假定有一个vector容器储存的arr，则

reverse（arr.begin(),arr.end()）

可以倒转整个vector容器

II.unique去重

作用是去除容器或数组中相邻元素的重复出现的元素，返回值是去重之后的尾地址

int m=unique（arr.begin(),arr.end()）-arr.begin() //m返回的是去重后的元素个数

需要重点注意的是，unique去重的是相邻元素的重复元素，而不是整个容器或数组中的重复元素，所有使用unique的时候一般要对整个容器或者数组进行sort排序（见下文）

III.sort排序

1.sort排序是类似于快速排序的一种排序方式，时间复杂度为n*log2(n),执行效率比较高，一般首选sort

2.sort的使用方法
sort含有三个参数sort（begin,end,cmp）,其中begin为指向待sort（）的数组的首元素的指针，end为指向待sort（）的数组的尾元素的下一位指针（不是尾元素，是尾元素的下一位！！）

cmp：自定义排序函数，返回bool类型，是一个布尔值函数

bool 是 C++ 的布尔类型，只有两个取值：

  • true（真，等价于 1）
  • false（假，等价于 0）

bool 函数的核心作用：返回判断结果（是 / 否、对 / 错）

# ✅ 正确写法

  • 升序：return a < b
  • 降序：return a > b

# ❌ 绝对不能写

  • return a <= b

  • return a >= b

  • bool 函数就是只返回 true 或 false

    a < b 本身，本来就是一个布尔表达式，若a < b，则true，否则返回false
